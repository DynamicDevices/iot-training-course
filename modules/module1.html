Day 3: Transport & Application Protocols

Session 5: UDP - Connectionless Protocol

75 minutes

UDP Characteristics:

  • Best effort delivery (no guaranteed delivery)
  • Connectionless (no connection establishment)
  • Low overhead (minimal protocol headers)
  • Use cases: Real-time data, streaming, sensor readings

ESP32 UDP Example:

// ESP32 UDP sensor transmission
WiFiUDP udp;
String data = String(temperature) + "," + String(humidity);
udp.beginPacket(server_ip, 8080);
udp.print(data);
udp.endPacket();

Session 6: TCP - Connection-Oriented Protocol

105 minutes

TCP Features:

  • Three-way handshake (SYN, SYN-ACK, ACK)
  • Reliable delivery with acknowledgments
  • Flow control and congestion control
  • Connection termination (FIN/ACK sequence)

TCP Three-Way Handshake:

1. Client ‚Üí Server: SYN (synchronize)
2. Server ‚Üí Client: SYN-ACK (synchronize-acknowledge)  
3. Client ‚Üí Server: ACK (acknowledge)

üî¨ Practical Exercise:

  • Monitor TCP connection states with Wireshark
  • Configure TCP socket options

90 minutes

HTTP Fundamentals:
