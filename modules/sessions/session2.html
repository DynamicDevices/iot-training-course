DHCP vs Static IP Configuration

DHCP (Dynamic Host Configuration Protocol)

DHCP Process (DORA):

  1. Discover: Client broadcasts request for IP
  2. Offer: Server offers available IP address
  3. Request: Client requests the offered IP
  4. Acknowledge: Server confirms IP assignment
AspectDHCPStatic IP
ConfigurationAutomaticManual
ManagementCentralizedPer-device
FlexibilityHighLow
ReliabilityDepends on serverIndependent
IoT SuitabilityDevelopment/TestingProduction

IoT Considerations

  • DHCP Reservations: Assign same IP to specific MAC
  • Lease Time: Longer leases for IoT devices
  • Fallback: Static IP if DHCP fails
  • Security: MAC filtering, separate VLAN

ARP (Address Resolution Protocol)

ARP Purpose

Maps IP addresses to MAC addresses on the local network segment.

ARP Process

  1. Device needs to send data to an IP address
  2. Checks ARP cache for MAC address
  3. If not found, broadcasts ARP request
  4. Target device responds with its MAC address
  5. Sender caches the MAC address for future use

ARP Message Types

  • ARP Request: "Who has IP x.x.x.x?"
  • ARP Reply: "IP x.x.x.x is at MAC yy:yy:yy:yy:yy:yy"
  • Gratuitous ARP: Announces own IP/MAC mapping

ARP Cache Management

View ARP Cache:

arp -a                    # Windows/Linux
ip neigh show             # Linux

Clear ARP Cache:

arp -d *                 # Windows
sudo ip neigh flush all   # Linux

Gateways and Routing

Default Gateway Role

  • Local Traffic: Direct communication within subnet
  • Remote Traffic: Forwarded to default gateway
  • Routing Decision: Based on destination IP and subnet mask

Routing Table Basics

View Routing Table:

route print              # Windows
ip route show            # Linux
netstat -rn              # Unix/Linux

Common Routing Entries

  • 0.0.0.0/0: Default route (all traffic)
  • 127.0.0.0/8: Loopback network
  • Local subnet: Direct delivery
  • Multicast: 224.0.0.0/4

Public vs Private IP Addresses

Private IP Ranges (RFC 1918)

10.0.0.0/8

Large networks (16.7M addresses)

Enterprise, ISP internal

172.16.0.0/12

Medium networks (1M addresses)

Corporate networks

192.168.0.0/16

Small networks (65K addresses)

Home, small office

NAT (Network Address Translation)

  • Purpose: Allow private IPs to access internet
  • Process: Router translates private to public IP
  • Port Mapping: Tracks connections by port numbers
  • IoT Impact: Devices not directly accessible from internet

Special IP Addresses

  • 127.0.0.1: Loopback (localhost)
  • 169.254.x.x: APIPA (Automatic Private IP)
  • 224.0.0.0-239.255.255.255: Multicast
  • 255.255.255.255: Limited broadcast

Practical Exercises

Exercise 1: DHCP Analysis

Release and renew your IP address while capturing with Wireshark:

  1. Start Wireshark capture with filter: dhcp
  2. Release IP: ipconfig /release
  3. Renew IP: ipconfig /renew
  4. Analyze the DORA process in captured packets

Exercise 2: ARP Investigation

Observe ARP behavior:

  1. Clear ARP cache: arp -d *
  2. Ping your gateway: ping [gateway-ip]
  3. Check ARP cache: arp -a
  4. Capture ARP traffic with Wireshark
