1. Introduction to Linux - Why It Matters for IoT

What is Linux and Why Should You Care?

Linux is a free, open-source operating system kernel originally created by Linus Torvalds in 1991 while he was a computer science student at the University of Helsinki. Think of it as the "brain" of the operating system that manages hardware resources and provides services to applications.

Why Linux Dominates IoT Development:

  • Cost-Effective: No licensing fees - crucial for IoT devices produced in millions
  • Customizable: You can strip it down to run on devices with just 32MB of RAM
  • Stable & Secure: Many IoT devices run for years without rebooting
  • Industry Standard: 90% of cloud infrastructure and most IoT gateways run Linux
  • Developer-Friendly: Excellent development tools and debugging capabilities
  • Hardware Support: Runs on everything from Raspberry Pi to industrial controllers
# Real-world example: Check what's running on your system
uname -a    # Shows kernel version and system information
cat /proc/version    # Detailed kernel information
lsb_release -a    # Distribution information (on Ubuntu/Debian)

# Example output might look like:
# Linux raspberrypi 5.15.84-v7l+ #1613 SMP Thu Jan 5 12:01:26 GMT 2023 armv7l GNU/Linux

Linux Distributions for IoT Development

A Linux distribution (or "distro") is a complete operating system built around the Linux kernel. Different distributions are optimized for different use cases. Here's what you need to know as an IoT developer:

Ubuntu/Debian Family

Development & Prototyping
Best For: Learning, development workstations, Raspberry Pi projects
Package Manager: apt (Advanced Package Tool)
Why Choose It: Excellent documentation, large community, easy to use
IoT Examples: Ubuntu Core for IoT gateways, Raspberry Pi OS for prototypes
Industry Use: Canonical's Ubuntu Core powers many commercial IoT devices

Red Hat Enterprise Linux (RHEL)/CentOS

Enterprise & Industrial IoT
Best For: Enterprise IoT deployments, industrial systems
Package Manager: yum/dnf (Yellowdog Updater Modified)
Why Choose It: Enterprise support, long-term stability, security focus
IoT Examples: Factory automation systems, critical infrastructure
Industry Use: Many Fortune 500 companies use RHEL for IoT infrastructure

Yocto/OpenEmbedded

Custom Embedded Systems
Best For: Production IoT devices, custom hardware
Package Manager: BitBake (build system)
Why Choose It: Highly customizable, minimal footprint, reproducible builds
IoT Examples: Smart thermostats, automotive systems, medical devices
Industry Use: Tesla, BMW, and many IoT manufacturers use Yocto 
# How to identify your Linux distribution
cat /etc/os-release    # Standard way to check distribution info
hostnamectl    # System information including OS
lscpu    # CPU architecture information

# Example: Setting up a development environment
# On Ubuntu/Debian:
sudo apt update && sudo apt upgrade    # Update package lists and system
sudo apt install build-essential git python3-pip    # Install development tools

# On RHEL/CentOS:
sudo yum update    # Update system
sudo yum groupinstall "Development Tools"    # Install development tools

2. Linux File System Hierarchy - Your Digital Map

Understanding the File System Structure

Unlike Windows with its C:, D: drives, Linux uses a single hierarchical tree structure starting from the root directory (/). Think of it like a family tree - everything branches out from one common ancestor. This design makes Linux systems more organized and predictable.

Key Concept: In Linux, everything is a file. This includes directories, devices (like your USB ports), processes, and even network connections. This unified approach makes system administration more consistent.

/                           # Root directory (the "trunk" of the tree)
├── bin/                    # Essential user binaries (commands like ls, cp, mv)
├── boot/                   # Boot loader files (kernel, initrd)
├── dev/                    # Device files (hardware interfaces)
│   ├── sda1               # First partition of first hard drive
│   ├── ttyUSB0            # USB serial device (common for IoT)
│   └── gpio/              # GPIO pins (on Raspberry Pi)
├── etc/                    # System configuration files
│   ├── network/           # Network configuration
│   ├── systemd/           # Service configuration
│   └── passwd             # User account information
├── home/                   # User home directories
│   ├── student/           # Your personal directory
│   └── iot/               # IoT service user directory
├── lib/                    # Essential shared libraries
├── media/                  # Removable media mount points (USB drives)
├── mnt/                    # Temporary mount points
├── opt/                    # Optional software packages
│   └── iot-platform/      # Custom IoT software installation
├── proc/                   # Process and kernel information (virtual filesystem)
│   ├── cpuinfo            # CPU information
│   └── meminfo            # Memory information
├── root/                   # Root user's home directory
├── run/                    # Runtime data (process IDs, sockets)
├── sbin/                   # System binaries (admin commands)
├── srv/                    # Service data
├── sys/                    # System information (virtual filesystem)
│   ├── class/gpio/        # GPIO control interface
│   └── class/thermal/     # Temperature sensors
├── tmp/                    # Temporary files (cleared on reboot)
├── usr/                    # User programs and data
│   ├── bin/               # User binaries
│   ├── lib/               # User libraries
│   ├── local/             # Local software installations
│   └── share/             # Shared data (documentation, icons)
└── var/                    # Variable data (logs, databases, caches)
    ├── log/               # System and application logs
    │   ├── syslog         # System log file
    │   └── iot/           # IoT application logs
    ├── lib/               # Variable state information
    └── cache/             # Application cache files

Key Directories Explained for IoT Developers

As an IoT developer, you'll work with these directories regularly. Here's what each one means for your daily work:

Essential commands (ls, cp, mv) and user programs. Your compiled IoT applications often go in /usr/local/bin
Configuration files for system and applications. IoT device settings, network configs, and service definitions live here
User directories. Your development projects, source code, and personal configurations are stored here
Log files for debugging. When your IoT application crashes, you'll look here first
Device files for hardware access. Serial ports (/dev/ttyUSB0), GPIO pins, and sensors appear here
Optional software. Many IoT platforms and third-party tools install here
# Practical examples for IoT developers

# Check available serial devices (common for IoT sensors)
ls -la /dev/ttyUSB* /dev/ttyACM*

# View system logs for troubleshooting
sudo tail -f /var/log/syslog    # Follow system log in real-time
sudo journalctl -u your-iot-service    # View logs for specific service

# Check network configuration
cat /etc/network/interfaces    # Network interface configuration
ip addr show    # Current network interfaces and IP addresses

# Monitor system resources (important for resource-constrained IoT devices)
cat /proc/cpuinfo    # CPU information
cat /proc/meminfo    # Memory usage
df -h    # Disk space usage

# GPIO access on Raspberry Pi
ls /sys/class/gpio/    # Available GPIO pins
echo 18 > /sys/class/gpio/export    # Enable GPIO pin 18
echo out > /sys/class/gpio/gpio18/direction    # Set as output

3. Path Concepts and Navigation - Your GPS for Linux

Absolute vs Relative Paths - Think Like a GPS

Understanding paths is like understanding addresses. An absolute path is like a complete street address that works from anywhere in the world. A relative path is like directions from your current location ("go two blocks north, then turn left").

Absolute Paths - The Complete Address

Always start with "/" (root directory) and specify the complete path. These work from anywhere in the system, just like a complete postal address.

# Examples of absolute paths (work from anywhere)
/home/student/iot-projects/sensor-data.csv
/etc/network/interfaces
/var/log/syslog
/usr/bin/python3
/opt/iot-platform/config/device.json

# Real-world IoT examples
/etc/systemd/system/iot-sensor.service    # Service configuration
/var/lib/iot/sensor-readings.db           # Database file
/home/iot/projects/temperature-monitor/   # Project directory

Relative Paths - Directions from Here

Start from your current location and use shortcuts. These are shorter but depend on where you are currently located.

# Relative path shortcuts - memorize these!
.          # Current directory (like "here")
..         # Parent directory (like "go up one level")
~          # Your home directory (like "go home")
-          # Previous directory (like "go back")

# Examples of relative paths
./sensor-data.csv              # File in current directory
../config/settings.json       # File in parent directory's config folder
~/projects/iot-gateway/        # Directory in your home folder
../../shared/libraries/        # Go up two levels, then down to shared/libraries

# Practical IoT development scenarios
cd ~/iot-projects              # Go to your IoT projects folder
./build.sh                     # Run build script in current directory
cp ../config/default.json .   # Copy config from parent directory to here
ls ../../shared/sensors/       # List sensors in shared directory

Essential Navigation Commands - Your Basic Toolkit

These commands are like the basic tools in a toolbox - you'll use them constantly. Practice until they become second nature.

# pwd - "Print Working Directory" (Where am I?)
pwd    # Shows your current location
# Example output: /home/student/iot-projects

# cd - "Change Directory" (Go somewhere)
cd /home/student              # Go to specific directory (absolute path)
cd ~/projects                 # Go to projects in your home directory
cd ..                         # Go to parent directory
cd -                          # Go back to previous directory
cd                            # Go to your home directory (shortcut)
cd iot-sensors/               # Go to iot-sensors subdirectory (relative path)

# ls - "List" (What's here?)
ls                            # List files in current directory
ls -l                         # Long format (shows permissions, size, date)
ls -la                        # Include hidden files (files starting with .)
ls -lh                        # Human-readable file sizes (KB, MB, GB)
ls /etc                       # List contents of /etc directory
ls *.py                       # List all Python files
ls -lt                        # Sort by modification time (newest first)

# Practical workflow example
pwd                           # Check where you are
ls -la                        # See what's in current directory
cd ~/iot-projects            # Go to your projects
ls -lh                       # List projects with file sizes
cd temperature-sensor/       # Enter specific project
pwd                          # Confirm you're in the right place

Pro Tips for Efficient Navigation

# Tab completion - your best friend!
cd /ho[TAB]                   # Completes to /home/
cd ~/proj[TAB]                # Completes to ~/projects/
ls /etc/net[TAB]              # Completes to /etc/network/

# Command history shortcuts
history                       # Show command history
!!                           # Repeat last command
!cd                          # Repeat last command starting with "cd"
Ctrl+R                       # Search command history (type to search)

# Multiple operations
ls -la; pwd; date            # Run multiple commands in sequence
cd ~/projects && ls -la      # Change directory AND list contents (only if cd succeeds)

# Useful aliases (shortcuts) - add these to your ~/.bashrc
alias ll='ls -la'            # ll becomes shortcut for ls -la
alias ..='cd ..'             # .. becomes shortcut for cd ..
alias projects='cd ~/iot-projects'  # Quick jump to projects directory

4. File Types and Naming Conventions - The Rules of the Road

Linux File Types - Understanding What You're Looking At

Linux identifies file types by the first character in the long listing (ls -l). This is crucial for understanding what you can do with each file.

Text files, source code, executables, images. Most files you work with daily. Example: sensor-data.csv, main.py
Folders that contain other files. You can "cd" into these. Example: /home/student/projects/
Shortcuts to other files/directories. Like Windows shortcuts. Example: /usr/bin/python -> python3.9
Hardware interfaces. Block devices (b) like hard drives, character devices (c) like serial ports. Example: /dev/ttyUSB0
Inter-process communication channels. Advanced topic for system programming.
Network communication endpoints. Used by applications to communicate over networks.
# Understanding ls -l output
ls -l /dev/ttyUSB0
# Output: crw-rw---- 1 root dialout 188, 0 Jan 15 10:30 /dev/ttyUSB0
#         ^
#         This 'c' means it's a character device (serial port)

ls -l ~/projects/
# Output: drwxr-xr-x 2 student student 4096 Jan 15 10:30 iot-sensors
#         ^
#         This 'd' means it's a directory

ls -l /usr/bin/python
# Output: lrwxrwxrwx 1 root root 9 Jan 15 10:30 /usr/bin/python -> python3.9
#         ^
#         This 'l' means it's a symbolic link pointing to python3.9

# Practical examples for IoT development
file sensor-data.csv         # Identify file type
file /dev/ttyUSB0           # Check device type
ls -la /sys/class/gpio/     # List GPIO device files

Naming Conventions and Rules - Best Practices for Professionals

Good naming conventions prevent problems and make your code more professional. These rules will save you hours of debugging.

Critical Rules to Remember:

  • Case Sensitive: "Config.txt", "config.txt", and "CONFIG.txt" are three different files
  • No Spaces in Names: Use underscores or hyphens instead
  • Hidden Files: Files starting with "." are hidden (like .bashrc, .gitignore)
  • No File Extensions Required: Linux doesn't rely on extensions like Windows
  • Special Characters: Avoid @, #, $, %, &, *, (, ), [, ], {, }, |, \, :, ;, ", ', <, >, ?, /
  • Length Limit: Filenames can be up to 255 characters (but keep them reasonable)
# Good filename examples for IoT projects
sensor_data_2024_01_15.csv           # Clear, descriptive, with date
temperature_monitor.py               # Descriptive function name
iot_device_config.json              # Clear purpose
backup-2024-01-15.tar.gz            # Backup with date
README.md                           # Standard documentation file
.env                                # Hidden environment variables file

# Bad filename examples (avoid these!)
"sensor data.txt"                   # Spaces require quotes in commands
sensor@data#2024.csv               # Special characters cause problems
verylongfilenamethatishard...       # Too long and unclear
data                               # Too generic, unclear purpose
CONFIG.TXT                         # Inconsistent case usage

# Professional IoT project structure example
iot-temperature-monitor/
├── src/
│   ├── main.py
│   ├── sensor_reader.py
│   └── data_processor.py
├── config/
│   ├── device_config.json
│   └── network_settings.yaml
├── logs/
│   └── application.log
├── tests/
│   └── test_sensor_reader.py
├── docs/
│   └── README.md
└── requirements.txt

# Working with files that have spaces (if you must)
ls "file with spaces.txt"          # Use quotes
mv "old name.txt" new_name.txt      # Rename to remove spaces
cp "sensor data.csv" sensor_data.csv  # Create copy with better name

Troubleshooting Common Issues

# Problem: "Permission denied" when accessing directories
# Solution: Check if you have the right permissions
ls -la /some/directory    # Check permissions
sudo ls -la /some/directory    # Use sudo if needed

# Problem: "No such file or directory"
# Solution: Check your path and spelling
pwd    # Where am I?
ls -la    # What's actually here?
ls -la /path/you/think/exists    # Verify the path exists

# Problem: Lost in the file system
# Solution: Use these commands to orient yourself
pwd    # Where am I?
cd ~    # Go home
ls -la    # What's in my home directory?

# Problem: Can't find a file you created
# Solution: Search for it
find ~ -name "filename"    # Search in home directory
find / -name "filename" 2>/dev/null    # Search entire system (ignore errors)

# Problem: Accidentally created file with spaces or special characters
# Solution: Use quotes or escape characters
ls -la "file with spaces.txt"
rm "file with spaces.txt"
# Or rename it:
mv "file with spaces.txt" file_with_underscores.txt
