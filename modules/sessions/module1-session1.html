1. Introduction to Linux

What is Linux?

Linux is a free, open-source operating system kernel originally created by Linus Torvalds in 1991. It forms the foundation of many operating systems (called distributions) that are widely used in servers, embedded systems, and IoT devices.

Key Characteristics:

  • Open Source: Source code is freely available and modifiable
  • Multi-user: Multiple users can work simultaneously
  • Multi-tasking: Can run multiple processes concurrently
  • Portable: Runs on various hardware architectures
  • Secure: Built-in security features and permission system
  • Stable: Reliable and rarely requires rebooting

Linux Distributions for IoT

2. Linux File System Hierarchy

Understanding the File System Structure

Linux uses a hierarchical file system that starts from the root directory (/). Everything in Linux is treated as a file, including directories, devices, and processes.

/                    # Root directory (top level)
├── bin/             # Essential user binaries
├── boot/            # Boot loader files
├── dev/             # Device files
├── etc/             # System configuration files
├── home/            # User home directories
│   ├── user1/       # Individual user directory
│   └── user2/       # Another user directory
├── lib/             # Essential shared libraries
├── media/           # Removable media mount points
├── mnt/             # Temporary mount points
├── opt/             # Optional software packages
├── proc/            # Process and kernel information
├── root/            # Root user's home directory
├── run/             # Runtime data
├── sbin/            # System binaries
├── srv/             # Service data
├── sys/             # System information
├── tmp/             # Temporary files
├── usr/             # User programs and data
│   ├── bin/         # User binaries
│   ├── lib/         # User libraries
│   └── local/       # Local software
└── var/             # Variable data (logs, databases)
    ├── log/         # System logs
    └── tmp/         # More temporary files

Key Directories Explained

Essential user commands (ls, cp, mv)
System configuration files
User home directories
Variable data (logs, databases)
User programs and libraries
Device files (hardware interfaces)

3. Path Concepts and Navigation

Absolute vs Relative Paths

Understanding paths is crucial for navigating the Linux file system effectively.

Absolute Paths

Start from the root directory (/) and specify the complete path to a file or directory.

# Examples of absolute paths
/home/user/documents/project.txt
/etc/network/interfaces
/var/log/syslog
/usr/bin/python3

Relative Paths

Start from the current working directory and use shortcuts to navigate.

# Relative path shortcuts
.          # Current directory
..         # Parent directory
~          # User's home directory
-          # Previous directory

# Examples of relative paths
./documents/project.txt    # File in current directory's documents folder
../config/settings.conf   # File in parent directory's config folder
~/Desktop/notes.txt       # File in user's Desktop folder

Basic Navigation Commands

Essential commands for moving around the file system:

# Print working directory (show current location)
pwd

# Change directory
cd /home/user          # Go to specific directory
cd ~                   # Go to home directory
cd ..                  # Go to parent directory
cd -                   # Go to previous directory
cd                     # Go to home directory (same as cd ~)

# List directory contents
ls                     # List files in current directory
ls -l                  # Long format (detailed information)
ls -la                 # Include hidden files (starting with .)
ls /etc                # List contents of /etc directory

4. File Types and Naming Conventions

Linux File Types

Linux recognizes several types of files, each serving different purposes:

Text, binary, executable files
Folders containing other files
Shortcuts to other files
Hardware device interfaces
Inter-process communication
Network communication

Naming Conventions and Rules

  • Case Sensitive: "File.txt" and "file.txt" are different files
  • Hidden Files: Files starting with "." are hidden (e.g., .bashrc)
  • No File Extensions Required: Linux doesn't rely on extensions like Windows
  • Special Characters: Avoid spaces and special characters in filenames
  • Length Limit: Filenames can be up to 255 characters long
  • Reserved Names: Avoid names like "." and ".."
# Good filename examples
project_report.txt
sensor_data_2024.csv
iot_device_config
backup-2024-01-15.tar.gz

# Avoid these patterns
"file with spaces.txt"    # Spaces require quotes
file@#$.txt              # Special characters can cause issues
CON, PRN, AUX            # Reserved names on some systems
