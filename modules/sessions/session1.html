Key Concepts:

  • Encapsulation: Each layer adds its own header
  • Abstraction: Upper layers don't need to know lower layer details
  • Modularity: Changes in one layer don't affect others

MAC Addresses and Physical Layer

MAC Address Structure

  • Format: 48-bit (6 bytes) hexadecimal
  • Example: 00:1B:44:11:3A:B7
  • OUI: First 3 bytes identify manufacturer
  • NIC: Last 3 bytes are device-specific

Physical Layer Considerations for IoT

  • Wi-Fi: 2.4GHz vs 5GHz bands
  • Range: Signal strength affects reliability
  • Interference: Other devices, microwaves, etc.
  • Power consumption: Critical for battery-powered devices

IP Address Classes and Subnetting

IPv4 Address Classes

Class A

Range: 1.0.0.0 - 126.255.255.255

Default Mask: 255.0.0.0 (/8)

Hosts: 16,777,214 per network

Class B

Range: 128.0.0.0 - 191.255.255.255

Default Mask: 255.255.0.0 (/16)

Hosts: 65,534 per network

Class C

Range: 192.0.0.0 - 223.255.255.255

Default Mask: 255.255.255.0 (/24)

Hosts: 254 per network

Private IP Ranges (RFC 1918)

  • Class A: 10.0.0.0/8 (10.0.0.0 - 10.255.255.255)
  • Class B: 172.16.0.0/12 (172.16.0.0 - 172.31.255.255)
  • Class C: 192.168.0.0/16 (192.168.0.0 - 192.168.255.255)

Network vs Device Identification

Understanding Network Portions

Example: 192.168.1.100/24

  • Network portion: 192.168.1.0 (first 24 bits)
  • Host portion: 100 (last 8 bits)
  • Subnet mask: 255.255.255.0
  • Network address: 192.168.1.0
  • Broadcast address: 192.168.1.255
  • Usable range: 192.168.1.1 - 192.168.1.254

Subnetting for IoT Networks

  • Segmentation: Separate IoT devices from other traffic
  • Security: Isolate devices with different trust levels
  • Management: Easier to monitor and troubleshoot
  • Performance: Reduce broadcast domains
